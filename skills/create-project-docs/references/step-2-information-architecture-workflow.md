# Step 2 Workflow: Design the doc split

## Objective

Place topics into `README.md` or `docs/references/` using explicit progressive-disclosure rules.

## Required actions

1. Put content in `README.md` when it is needed for:
   - first successful setup
   - first successful run
   - understanding the project's top-level purpose
   - understanding the project's core nouns
2. Move content into `docs/references/` when it is:
   - schema-heavy
   - exhaustive
   - table-driven
   - role-specific
   - caveat-heavy
   - best explained with multiple examples
3. Assign each justified deeper topic one archetype:
   - workflow guide
   - walkthrough
   - catalog
   - lexicon
   - comparison memo
4. Reject planned deeper docs that have no distinct purpose beyond repeating the README.
5. Record one placement reason for every major topic so the split is reviewable.

## Done when

- Each major topic has a documented placement decision.
- Each deeper doc has a defined archetype and purpose.
- The README scope stays concise enough to read top-to-bottom.
