# Step 5 Workflow: Select templates and examples

## Objective

Use the skill's packaged templates and example snapshots to keep outputs consistent without copying source material blindly.

## Required actions

1. Use `../assets/templates/project-readme-template.md` as the default README structure.
2. Select the matching deeper-doc template for each justified topic:
   - `../assets/templates/reference-catalog-template.md`
   - `../assets/templates/reference-comparison-template.md`
   - `../assets/templates/reference-lexicon-template.md`
   - `../assets/templates/reference-walkthrough-template.md`
   - `../assets/templates/reference-workflow-template.md`
3. Read only the example files that match the archetypes you plan to generate:
   - `../examples/raidar-readme-example.md`
   - `../examples/raidar-auto-researcher-reference-example.md`
   - `../examples/raidar-env-var-lexicon-example.md`
   - `../examples/raidar-framework-comparison-example.md`
   - `../examples/raidar-homepage-walkthrough-example.md`
   - `../examples/raidar-metrics-catalog-example.md`
   - `../examples/raidar-new-harness-example.md`
   - `../examples/raidar-new-scenario-example.md`
   - `../examples/raidar-orchestration-flow-example.md`
4. Treat examples as pattern references for:
   - section ordering
   - topic depth
   - when to move content out of the README
   - example density
5. Do not copy example commands, paths, names, or repo-specific terminology into the target repo unless the target repo evidence independently justifies them.
6. Prefer the template when examples and target-repo evidence are in tension.

## Done when

- Every drafted file uses a local template.
- Example files influenced structure and flow without overriding repo evidence.
- No target-repo output contains example-repo specifics by accident.
