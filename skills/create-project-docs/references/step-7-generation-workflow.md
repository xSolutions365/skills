# Step 7 Workflow: Generate files and report gaps

## Objective

Write the approved docs into the target repo and hand back a concise evidence-backed summary.

## Required actions

1. Write or update `README.md` in the target repo exactly as approved.
2. Write or update only the justified `docs/references/*.md` files.
3. Preserve existing files that are outside the approved documentation scope.
4. Report:
   - created files
   - updated files
   - evidence sources used
   - unresolved gaps or contradictions
5. If the repo's evidence remains incomplete, keep the gap visible in the handoff instead of fabricating extra documentation.

## Done when

- Approved documentation files are written.
- The handoff identifies what changed and why.
- Any unresolved uncertainty is explicit and actionable.
