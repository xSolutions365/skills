# Step 3 Workflow: Draft the README

## Objective

Generate a concise README that provides fast onboarding without burying specialist detail.

## Required actions

1. Start from [../assets/templates/project-readme-template.md](../assets/templates/project-readme-template.md).
2. Keep these sections in order:
   - title
   - badges
   - `Quick Install`
   - `Quick Start`
   - `What <Project> Does`
   - `Core Concepts`
   - `Go Deeper`
3. Write one plain-English value statement before introducing internal jargon.
4. Use one canonical install path and one canonical quick-start path. Add at most one advanced example only if it clarifies the public interface materially.
5. Keep badges factual. Good badge categories include status, language/runtime, primary CLI, package manager, or platform support when the repo evidence supports them.
6. Link only to deeper docs that the Step 2 split justified.
7. Reject any README content that invents commands, paths, environment variables, or behavior.

## Done when

- The README stands on its own as an entrypoint doc.
- A new contributor can find setup, first run, project purpose, and core concepts quickly.
- The README points to deeper docs instead of absorbing all detail itself.
