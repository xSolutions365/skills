# Prose Style Guide

## Objective

Set the universal prose rules that apply across all presets and both create and redraft modes.

## Required actions

1. Write as someone accountable for the outcome, not as someone narrating a method.
2. Bind the prose to the actual context. If the draft would still work after a subject-name swap, rewrite for specificity.
3. Treat the piece as one argument with visible progression, not as disconnected competent sentences.
4. Prefer concrete nouns, real constraints, and observable consequences over generic value language.
5. Choose a layout shape that fits the job. Do not default to uninterrupted paragraphs when a short flat list or compact comparison would make the point clearer.
6. Vary sentence openings, sentence lengths, and paragraph lengths enough to avoid mechanical rhythm.
7. Remove prestige filler, brochure tone, and empty strategic language.
8. Do not open by explaining what the section will do.
9. Do not use capability lists unless each item is tied directly to the current need.
10. When concrete detail is unavailable, state that limit once instead of padding with abstractions.
11. Do not end with a decorative summary paragraph that only restates earlier points.

## Banned terms and phrases

Do not use these unless they are part of quoted source text:

- Hollow understanding statements: `we understand your challenge`, `we recognise the complexity`
- High-confidence AI-tell vocabulary: `delve`, `multifaceted`, `pivotal`, `holistic`, `underscores`, `realm`, `game-changer`
- Generic credibility claims: `world class`, `best practice`, `best-in-class`, `state-of-the-art`, `proven track record`, `deep expertise`
- Hollow outcome language: `deliver value`, `drive transformation`, `enable success`, `unlock potential`
- Section-purpose openers: `this section outlines`, `the purpose of this piece is`
- Generic framing openers: `the engagement is designed to`, `this report will`

## Anti-pattern constraints

Do not use these patterns:

- Opening by describing the section purpose
- Capability lists without context linkage
- Closing paragraph that only restates prior points
- Consecutive paragraphs with the same sentence structure or similar length
- Three or more consecutive sentences starting with the same word
- Mechanical transition phrases as paragraph crutches: `furthermore`, `moreover`, `additionally`

## Done when

- The draft feels written for this exact context.
- The prose has a visible line of thought.
- Tone is direct, specific, and unsentimental.
- No paragraph reads like filler or generic positioning.
- The structure looks chosen, not defaulted.
