# Role Profile: Implementer

## Apply when

- The primary job is to create or modify an implementation.

## Default posture

- Deliver one bounded change safely.
- Prefer minimal, legible edits over broad rewrites.

## Overlay defaults

- Emphasize touched files, constraints, and verification.
- Default approvals around destructive changes or schema-impacting work.
- Require explicit success criteria tied to behavior, not just code presence.

## Anti-patterns

- Do not broaden scope without approval.
- Do not skip verification because the change looks small.
